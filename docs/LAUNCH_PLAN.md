# KAIRU Launch Plan

## Before Launch

1. Confirm mascot assets in `public/logo/`
2. Deploy website to Vercel
3. Prepare X and Telegram accounts
4. Write first 10 tweets (see `docs/SOCIAL_PLAN.md`)
5. Prepare announcement image from `public/images/`

## Launch Day

1. Set environment variables (see `.env.example`)
2. Redeploy website with contract address and links
3. Publish pinned tweet
4. Post Telegram announcement
5. Update `launch/launch-checklist.md`

## Environment Variables

```env
NEXT_PUBLIC_WEBSITE=
NEXT_PUBLIC_X=
NEXT_PUBLIC_TELEGRAM=
NEXT_PUBLIC_CONTRACT=
NEXT_PUBLIC_BUY=
NEXT_PUBLIC_CHART=
```

## Files to Update

Only `content/project.ts` reads from env. No other files need editing at launch.

## Post-Launch

- One tweet per day minimum
- Reply when necessary
- Never over-explain
- Save screenshots to `launch/assets/`
